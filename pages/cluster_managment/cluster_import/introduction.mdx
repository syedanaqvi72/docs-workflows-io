### Import existing Kubernetes Cluster

Gripo also provides the facility to import an existing kubernetes cluster from different cloud providers into the Gripo Portal. Follow the articles to import your existing kubernetes cluster from different cloud provider to Gripo.

| Cloud Provider        | Import |
| ---------------------- | ----------- |
| AWS EKS          | [EKS Cluster](/cluster_managment/cluster_import/amazon) |
| Google Kubernetes Engine (GKE) | [GKE Cluster](/cluster_managment/cluster_import/google) |
| Azure Kubernetes Service (AKS) | [AKS Cluster](/cluster_managment/cluster_import/azure). |
| DigitalOcean Kubernetes Service (DigitalOcean) | [DigitalOcean Cluster](/cluster_managment/cluster_import/digital-ocean). |
| Linode Kubernetes Service (LKE) | [Linode Cluster](/cluster_managment/cluster_import/linode). |


#### Navigating to the Cluster Import Section
* Navigate to the <b>"Management" </b>tab on the dashboard.
* Within the "Management" section, you will find a button labeled <b>"Cluster".</b> Click on this button to initiate the cluster creation process
* In the upper-right corner, click the <b>"Import"</b> button.
<br />
![Import-Cluster](/import_cluster/import-cluster0.png)<br />
* When you click on Import, a pop-up window appears presenting two options: Import by Secret and Import by Config File. The user can select their preferred option based on their requirements.
