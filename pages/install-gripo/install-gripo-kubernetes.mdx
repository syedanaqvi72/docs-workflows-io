### Setting Up Docker, Helm, Kind, and Kubectl on Windows for Deploying Gripo

This guide will walk you through the steps to set up a Kubernetes environment on Windows using Docker Desktop, Helm, Kind, and Kubectl, and then install<b> Gripo</b> on the Kind cluster.

Here are the direct links:
 
* [Helm](https://github.com/helm/helm/releases)


* [ Helm](https://github.com/helm/helm/releases)
* [ Kubectl](https://kubernetes.io/docs/tasks/tools/install-kubectl-windows/)
* [ Kind](https://kind.sigs.k8s.io/docs/user/quick-start/)
* [Github Desktop:](https://desktop.github.com/download/)

###### Install Docker Desktop
To begin, install Docker Desktop on your Windows machine by following the official installation guide:<br />
[Docker Desktop Installation Guide](https://docs.docker.com/desktop/setup/install/windows-install/)
<br />
Once installed, Docker Desktop enables containerized environments on your system, which is essential for running Kubernetes.
<br />
<b>Fixing WSL Error (If Encountered)</b>
After installing Docker Desktop, you might encounter an error related to WSL (Windows Subsystem for Linux). To resolve this:
<br />
<b>Enable Windows Features:</b>
* Open Control Panel > Programs > Turn Windows features on or off.
* Enable:
   - Windows Subsystem for Linux (WSL)
   - Virtual Machine Platform
   - Hyper-V 
* Restart your computer for changes to take effect.
##### Install Helm, Kind, and Kubectl on Windows
These tools are required for managing and running Kubernetes clusters locally.
<b>Install Helm</b>
* Helm is a package manager for Kubernetes, but Windows does not provide a direct installation package. Instead, you need to download it manually:

[ Download Helm for Windows](https://github.com/helm/helm/releases)
* Extract the downloaded ZIP file.
* Add the extracted folder to the system PATH so you can use helm from the command line.
<b>Install Kubectl</b>
* Kubectl is the command-line tool for interacting with Kubernetes clusters.

[ Download Kubectl for Windows](https://kubernetes.io/docs/tasks/tools/install-kubectl-windows/)
* Download the <b>.exe file.</b>
* Move it to a suitable location (e.g., C:\kubectl\).
* Add the directory path to the <b>system PATH.</b>
<b>Install Kind</b>
* Kind (Kubernetes in Docker) allows you to run Kubernetes clusters using containers.

[Kind Installation Guide](https://kind.sigs.k8s.io/docs/user/quick-start/)
* Download the <b>kind-windows-amd64</b> file.
* Rename the file to kind.exe.
* Move it to a convenient directory <b>(e.g., C:\kind\).</b>
* Add the directory path to the <b>system PATH.</b>
 ##### Create a Kind Cluster


Now that Kind is installed, you can create a local Kubernetes cluster.<br />
<b>Navigate to Kind Executable Location</b>
Open Command Prompt (cmd) or PowerShell, then navigate to the folder where kind.exe is located:
```sh
cd C:\kind\
```
<b>Create a Kubernetes Cluster with Kind</b>
* Run the following command to create a Kubernetes cluster:

```sh
kind create cluster
```
This command will set up a local Kubernetes cluster inside Docker.

##### Install Gripo Using Helm in Kind
<b>Add the Gripo Helm Repository</b>
Once your Kind cluster is running, add the Gripo Helm repository:
```sh
helm repo add gripo-io https://charts.gripo.io/
```
This command registers the Gripo chart repository, allowing you to install the package.
##### Install Gripo
Now, install the Gripo application in your Kubernetes cluster using Helm:

```sh
helm install gripo gripo-io/gripo
```
This command deploys Gripo in your Kind Kubernetes cluster.<br />
If you can't access the portal, run the following command again:
```sh
kubectl port-forward svc/gripo-nginx 5055:5055 --namespace default
```
<br />
##### Final Thoughts
After completing these steps, your Windows machine will be ready with a local Kubernetes environment running Gripo. You can verify the installation by running:
```sh
kubectl get pods
```
This command lists all running pods in your cluster. If everything is set up correctly, you should see Gripo running.
<b>For additional troubleshooting, refer to:</b>


* [Helm Documentation:](https://helm.sh/docs/)
* [Kind Documentation:](https://kind.sigs.k8s.io/) 

* [Kubernetes Documentation:](https://kubernetes.io/docs/)
This guide ensures that you have a functional Kubernetes environment using Kind, and that you have successfully deployed Gripo.

    ### Prerequisite

Ensure you have the following installed:<br />

* Docker (for container runtime)
* [Helm](https://github.com/helm/helm/releases)
* [Kubectl](https://kubernetes.io/docs/tasks/tools/install-kubectl-windows/)

### Set Up a Local Kubernetes Cluster
<br />
<b>Using Kind</b>

Create a new Kind cluster:

```sh
kind create cluster --name gripo-cluster
```

Check if the cluster is running:

```sh
kubectl cluster-info --context kind-gripo-cluster
```
<br />

<b>  Load the Helm Chart</b>


If the chart is hosted in a Helm repository:

```sh
helm repo add gripo-io https://charts.gripo.io
helm repo update
helm install gripo gripo-io/gripo
```
<br />

<b>Verify Deployment</b> 

Check if the pods are running: 

 ```sh 
kubectl get pods
```
To check Helm release:

 ```sh 
helm list
```

### Access the gripo portal

Depending on your deployment type  to use  Kind follow the below instructions: 
<br />
<b>Kind (Using Port Forwarding)</b>

Forward the port:
<br />
 ```sh 
kubectl port-forward svc/gripo-nginx 5055:5055 --namespace default
```
<br />
 Now access http://localhost:8080 in your browser.

<br />
<b> Customize Deployment(Optional)</b>

Fetch the Values YAML File If you want to save the values to a file for editing:

```sh 
helm show values gripo-repo/example-chart > custom-values.yaml
```

Modify values using custom custom values.yaml:

 ```sh 
helm install gripo -f custom-values.yaml gripo-io/gripo
```
<br />
### Stop and Cleanup
<br />
<b>Stopping the Cluster</b>

* For Kind:

 ```sh 
kind delete cluster --name gripo-cluster
```
<br />
<b>Uninstall Helm Release</b>
 
```sh 
helm uninstall gripo
```
